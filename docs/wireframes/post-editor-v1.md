# 게시글 에디터 와이어프레임 v1

## 개요

- 목적: 작성자가 제목/본문/발행 액션을 혼동 없이 수행하도록 작성 흐름을 단순화
- 타깃: 로그인 사용자(작성 권한 보유)
- 톤: 집중형 작업 화면, 기능 우선 레이아웃

## 화면 구조

```text
┌──────────────────────────────────────────────────────────────┐
│ Sticky Header: 뒤로가기 | 화면 제목 | 임시저장 상태          │
├──────────────────────────────────────────────────────────────┤
│ Category                                                     │
│ [자유수다 ▼]                                                 │
├──────────────────────────────────────────────────────────────┤
│ Title                                                        │
│ [제목 입력 필드...........................................]  │
├──────────────────────────────────────────────────────────────┤
│ Body Editor                                                  │
│ [툴바]                                                       │
│ [본문 작성 영역...........................................]  │
│ [이미지 업로드]                                              │
├──────────────────────────────────────────────────────────────┤
│ Helper Zone                                                  │
│ - 작성 가이드 / 글자 수 / 에러 메시지                        │
├──────────────────────────────────────────────────────────────┤
│ Bottom Action Bar                                            │
│ [취소]                                   [임시저장] [등록]   │
└──────────────────────────────────────────────────────────────┘
```

## 컴포넌트 매핑

- 폼 필드: Label, Input, Textarea
- 카테고리: Select(또는 커스텀 선택 컴포넌트)
- 하단 액션: Button(ghost, outline, default)
- 경고/오류: Alert 스타일 블록

## 상태 표현

- 로딩: 기존 글 불러오는 중 안내
- 빈 상태: 초안 없음 기본 안내
- 오류: 저장 실패 토스트 + 재시도 액션

## 반응형 규칙

- 모바일: 하단 액션 바 버튼 2행 허용
- 태블릿 이상: 액션 버튼 한 줄 정렬
- 데스크톱: 에디터 높이 최소 60vh 유지
