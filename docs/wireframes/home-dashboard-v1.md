# 홈 대시보드 와이어프레임 v1

## 개요

- 목적: 방문자가 첫 화면에서 블로그 정체성과 핵심 동선(읽기/작성)을 즉시 이해하도록 설계
- 타깃: 블로그 독자, 글 작성자
- 톤: 다크 우선, 여백 중심, 편집형 카드 레이아웃

## 화면 구조

```text
┌──────────────────────────────────────────────────────────────┐
│ Header: Brand | Home | Posts | Hobby | Goals | MyPage      │
├──────────────────────────────────────────────────────────────┤
│ Hero                                                    [CTA]│
│ - 한 줄 소개                                               │
│ - 보조 설명                                                │
├──────────────────────────────────────────────────────────────┤
│ Metrics Row: [총 글 수] [최근 업데이트] [댓글 수] [태그 수] │
├──────────────────────────────────────────────────────────────┤
│ Featured Grid                                              │
│ [Card A] [Card B] [Card C]                                 │
│ [Card D] [Card E] [Card F]                                 │
├──────────────────────────────────────────────────────────────┤
│ Secondary CTA: 최신 글 더 보기 / 글 작성하기                │
├──────────────────────────────────────────────────────────────┤
│ Footer: GitHub | Instagram | Contact                       │
└──────────────────────────────────────────────────────────────┘
```

## 컴포넌트 매핑

- 상단 내비게이션: MainLayout + 링크 그룹
- 히어로 액션: Button
- 지표 묶음: Card
- 추천 콘텐츠: Card, Badge
- 보조 액션: Button variant="outline"

## 상태 표현

- 로딩: 카드 스켈레톤 6개
- 빈 상태: 추천 콘텐츠 없음 안내 + 작성 버튼
- 오류: 피드 로드 실패 안내 + 재시도 버튼

## 반응형 규칙

- 모바일: 히어로 1열, 카드 1열
- 태블릿: 카드 2열
- 데스크톱: 카드 3열

## 생성 프롬프트(기록)

```text
Create a dark-first personal blog home wireframe for Next.js App Router.
Use a centered hero, editorial card grid, and one prominent primary CTA.
Keep the layout clean, spacious, and responsive.
```
