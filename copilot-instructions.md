# Copilot 지침

## 프로젝트 개요

- 프로젝트: my-first-web
- 스택: Next.js 16.2.1, React 19, TypeScript, Tailwind CSS v4, Supabase
- 라우팅: App Router 전용
- UI 시스템: shadcn/ui를 초기화 이후 기본 컴포넌트 레이어로 사용
- 렌더링 규칙: 기본은 Server Component, 상태/이펙트/브라우저 API가 필요할 때만 Client Component 사용

## 디자인 토큰

테마의 단일 기준점은 CSS 변수로 유지한다. 인터페이스는 다크 우선 팔레트와 따뜻한 강조색을 통해 차분하고 읽기 쉬운 편집형 분위기를 지향한다.

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| background | #0f1217 | 앱 셸 배경 |
| surface | #151b23 | 카드, 패널, 떠 있는 영역 |
| surface-2 | #1b2230 | 보조 표면, 오버레이 |
| primary | #c29a73 | 주요 CTA, 활성 상태, 브랜드 강조 |
| primary-foreground | #18110b | primary 배경 위 텍스트 |
| text | #f4efe8 | 기본 텍스트 색상 |
| muted | #9aa3ad | 보조 텍스트, 헬퍼 카피 |
| border | #2a3340 | 테두리, 구분선, 입력 요소 |
| accent | #6da7ff | 링크, 포커스 강조, 정보 상태 |
| danger | #ef6b6b | 파괴적 액션, 오류 상태 |

타이포그래피 규칙:

- 제목은 정보 위계가 드러나도록 압축감과 높은 대비를 유지한다.
- 본문은 작은 화면에서도 읽기 쉽도록 충분한 가독성을 보장하고 과도한 줄 간격 압축을 피한다.
- 모노스페이스는 코드, ID, 타임스탬프, 기술 메타데이터에 한정한다.

간격 규칙:

- 여유 있는 섹션 구성과 일관된 세로 리듬을 우선한다.
- 카드, 버튼, 입력, 배지의 모서리 둥글기는 일관되게 유지한다.
- 시각적 분리 없이 긴 텍스트 덩어리가 생기지 않도록 한다.

## 컴포넌트 규칙

### 기본 시스템

- 일회성 커스텀 UI를 만들기 전에 shadcn/ui 컴포넌트를 우선 사용한다.
- shadcn init 이후 생성되는 프리미티브는 components/ui 영역에 모아 관리한다.
- 클래스 조합이 복잡해지면 문자열 덧붙이기 대신 cn 헬퍼를 사용한다.
- 반복되는 페이지 패턴은 마크업 복붙 대신 재사용 컴포넌트로 감싼다.

### 우선 사용 컴포넌트

- 모든 액션과 CTA는 Button 사용
- 표면/요약 레이아웃은 Card, CardHeader, CardContent, CardFooter 사용
- 폼 컨트롤은 Input, Textarea, Label 사용
- 카테고리/상태/메타데이터는 Badge 사용
- 그룹 전환은 Tabs 사용
- 상호작용이 많은 UI는 Dialog, Sheet, DropdownMenu 사용
- 일시적 피드백은 Toast 또는 Sonner 스타일 사용

### 레이아웃 규칙

- 헤더, 콘텐츠, 푸터를 포함한 단일 앱 셸 구조를 유지한다.
- 데스크톱에서는 읽기 좋은 최대 폭으로 중앙 정렬한다.
- 작은 화면의 보조 내비게이션은 드로어 또는 시트로 접는다.
- 공통 셸로 해결 가능한 레이아웃 책임을 페이지별로 중복 분산하지 않는다.

### 데이터/상태 규칙

- 데이터 접근은 lib 또는 기능 단위 훅 레이어에 둔다.
- 이미 헬퍼가 있는 경우 표시 전용 컴포넌트에서 Supabase를 직접 호출하지 않는다.
- 로딩, 빈 상태, 오류 상태를 명시적으로 드러낸다.
- 낙관적 UI는 롤백 시나리오가 명확할 때만 사용한다.

### 인터랙션 규칙

- 내비게이션 API는 next/navigation만 사용하고 next/router는 사용하지 않는다.
- App Router에서는 라우트 값을 읽기 전에 params를 await 한다.
- 접근성 라벨, 키보드 포커스, 시각적 포커스 링을 기본값으로 유지한다.
- 대상이 확정되고 사용자 의도가 확인되기 전에는 파괴적 액션을 비활성화한다.

## AI 응답 규칙

Copilot이 코드 변경 요청을 받으면 아래 형식으로 응답한다.

1. 변경한 파일 목록
2. 파일별 변경 이유
3. 결과 검증 방법
4. 후속 위험 요소 또는 엣지 케이스

## 피해야 할 대표 실수

- Pages Router를 사용하지 않는다.
- next/router를 사용하지 않는다.
- App Router의 params를 동기 방식으로 읽지 않는다.
- 명시적 요청이 없으면 두 번째 스타일링 시스템을 도입하지 않는다.
- 디자인 토큰으로 대체 가능한 자리에서 임의 색상을 추가하지 않는다.
